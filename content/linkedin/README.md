# JRS LinkedIn Content Store

This directory is the human-review staging area for LinkedIn content generated by the
Research Findings Engagement Engine. **Nothing here is auto-posted.** An administrator
copies, edits, and publishes manually.

## How content is generated

Every time `api/run-study.js` runs (manually or via the nightly Vercel Cron), it publishes
one finding to Supabase and, on the same row, generates a full discussion bundle:

| Field | Purpose |
|---|---|
| `observation` | The finding framed as an observation, not a conclusion |
| `research_question` | The open question the data raises |
| `discussion` / `debate` / `challenge` | Comment-generation prompts |
| `poll` (jsonb) | Poll question + options |
| `classification` | Observation → Emerging Pattern → Developing Trend → Supported → Replicated (by replication count) |
| `linkedin_short` | Short post (observation + question + link) |
| `linkedin_long` | Long post (observation, caveat, open question, CTA) |

## Where to get the copy-ready text

The generated LinkedIn text is surfaced for one-click copy on the finding page:

- **Latest finding:** https://www.jrsstandard.com/finding.html
- **Specific finding:** https://www.jrsstandard.com/finding.html?id=<finding-uuid>

Open the page → "Share This Finding" → copy the Short Post, Long Post, or Poll.

## Post templates (reference)

1. **Observation** — early observation + interpretive question + dashboard link.
2. **Challenge** — short record excerpt + "what's missing?" + One-Minute Challenge link.
3. **Poll** — "Which documentation issue creates the most problems?" A/B/C/D + "vote and explain".
4. **Dashboard** — Research Activity screenshot + "what finding would you expect?" + link.

## Guardrails (do use / do not use)

- **Do:** curiosity, professional disagreement, research questions, observations, real examples.
- **Do not:** sensational claims, hype, fear language, risk inflation, unsupported conclusions.
- Reproducibility ≠ accuracy ≠ validation. Keep that distinction in every post.

## Archiving published posts (optional)

To keep a record, save each posted variant here as `YYYY-MM-DD-<slug>.md` with the final
edited text and the LinkedIn URL. This keeps a manual audit trail alongside the Supabase row.
