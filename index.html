Document I · Scope

A Control Applied Before Any Record Is Finalized

JRS is applied to records before they become the system of record. AI-assisted drafting influences documentation. The resulting record carries evidentiary risk. JRS is a pre-submission control used before finalizing a record. It ensures reasoning remains clear, observable, and explainable when reviewed later without context.

Standard Requirement

The Standard

Stand on its own.

Be based on something observable.

Be clearly explainable.

If it does not stand on its own, do not finalize it.

Verification Check

The Check

  1. Standalone
    Does it stand on its own?
  2. Basis
    What is it based on?
  3. Independence
    Would someone else understand it without context?

If not, it is not ready.

Control Enforcement

The Simple Version

Control Does this meet the standard?
If it does not, revise it before it becomes the record.

Failure Example

When the Record Does Not Stand On Its Own

Failure Example

A record stating an employee is "not a strong cultural fit" does not stand on its own, is not based on observable criteria, and cannot be clearly explained without additional context. It is not ready for finalization.

Pre-Submission Checklist

Apply Before Saving Any Record

This standard is applied before finalizing records. Organizations incorporate it into internal review or audit processes. Each checkpoint reflects what is required before the record is saved.

```
  1. Standalone Clarity
    Read this record as if you were not the person who wrote it. Does it hold together without you in the room?
  2. Observable Basis
    Can you point to a specific document, feedback record, metric, or observed event for every claim in this record?
  3. Explainability
    Can you explain this record plainly to someone who asks what it is based on? If you need context that is not in the record, the record is not ready.
  4. Phrase Review
    Does this record contain language that is vague, impression-based, or not grounded in a defined, observable standard? If yes, replace it before saving.
  5. Accommodation Status
    If this record involves an employee with a known or disclosed disability or accommodation request, does it note the current status?
Severity Tier Standard records: apply all five checkpoints.
High-impact decisions (termination, demotion, hiring rejection): enhanced review required. Add a second reviewer.
Higher-risk decisions require additional independent review before finalization. The tier reflects the level of independent review warranted before the file is finalized.
All Pass
Submit the record.
The record is grounded, stands on its own, and reflects independent human judgment.
Any Fail
Revise before saving.
Do not finalize a record that cannot answer: what, specifically, is this based on?

Time spent before finalization reduces the need for post hoc explanation. Records are evaluated based on whether reasoning is supported by documented evidence.

```

JRS establishes a contemporaneous record of reasoning at the moment of creation. Courts and regulators rely on contemporaneous documentation when evaluating whether a decision is supported by evidence.

Regulatory Alignment

Colorado SB 24-205 (effective June 30, 2026) provides an affirmative defense for organizations demonstrating reasonable care through documented AI governance. JRS supports evidence of reasonable care. The EU AI Act (August 2, 2026) requires transparency documentation and human oversight for all high-risk AI used in employment decisions.

Section 03

Application Protocol

JRS is applied before finalizing any written decision.

```
  1. Read it back
    Read the file as if you were not in the room. Does it hold together without your context?
  2. Apply the three questions
    Does it stand on its own? Is it based on something observable? Can it be clearly explained if someone asks?
  3. Check for pattern phrases
    Does it contain any phrases from the risk reference? If yes, replace them before saving.
  4. Confirm the basis
    Can you point to a specific document, feedback record, metric, or observed behavior for every claim?
  5. Save or revise
    If all four steps pass, save. If any step fails, revise. Do not finalize a record that cannot answer: what, specifically, is this based on?
JRS is applied consistently, before every record, regardless of whether the decision feels difficult or routine.
```

Section 12

Chain of Reasoning Audit

For high-stakes decisions: terminations, demotions, and any decision where AI-assisted reasoning carries significant risk.

```
  1. The Original Decision
    Can the decision-maker state the basis for this decision without reference to anything AI produced? If no, the independence of the reasoning cannot be established.
  2. The Prompt History
    What was the AI asked to do? If the prompt contained the conclusion, the record built from that prompt is not independent reasoning.
  3. The Evidence File
    Does a file exist containing the evidence the documentation references? If the evidence does not exist outside the record, the file is not grounded.
  4. The Record Itself
    Apply the three-question check. Does it stand on its own? Is it based on something observable? Can it be clearly explained?
  5. The Second Review
    For terminations and demotions, has a second reviewer independent of the decision-maker reviewed the file before it was finalized?
Audit Observation In a recent termination case, the file appeared complete. Applying the audit, the decision-maker could not state the rationale without referring back to AI-generated text. The prompt had been shaped to produce the conclusion. No independent evidence file existed. The record did not pass Steps 01 through 03, and the decision was reopened before finalization.
Audit Observation In a separate disciplinary matter, the audit surfaced a different issue. Steps 01 and 02 passed, but the evidence file referenced events that did not exist outside the AI-generated narrative. The reasoning was independent; the supporting record was not. Step 03 failed, and the file was returned for documentation before any action was taken.

Records are evaluated based on whether reasoning is supported by documented evidence.

Closing

Limitations of JRS

JRS improves documentation quality. It also has limitations. The following identify limitations of the standard.

```
  1. Compliance Fatigue
    When managers use AI to rewrite records to meet JRS criteria, the AI produces language that looks defensible even when the underlying reasoning has not changed. JRS requires independent human judgment, not AI-polished compliance.
  2. Prompt Integrity
    JRS addresses what gets written. It does not address what drove the writing. A well-worded record built on a biased prompt is still a biased record.
  3. Shadow AI
    Many managers draft in personal AI accounts and paste into official platforms. The drafting is the creation. The standard applies at the moment of drafting, not just saving.
  4. Qualitative Over-Correction
    Observable does not mean a number on a spreadsheet. Documented failures and conduct must be captured accurately. Documentation must remain accurate and reflect actual events, not sanitized into numbers that obscure what actually happened.
  5. Safe Harbor Limitations
    JRS supports evidence of reasonable care. It does not provide complete legal protection. For decisions as significant as a termination or demotion, consult qualified employment counsel.
